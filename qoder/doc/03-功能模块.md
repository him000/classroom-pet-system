# 功能模块详解

## 1. 用户认证模块

### 1.1 登录系统

**功能描述**：支持三种角色的登录认证

**登录流程**：
```
输入账号密码 → 验证身份 → 跳转对应角色首页
```

**账号类型**：

| 角色 | 账号示例 | 密码 | 权限级别 |
|------|---------|------|---------|
| 管理员 | admin | admin888 | 最高权限 |
| 教师 | teacher | teacher123 | 班级管理权限 |
| 学生 | xiaoming | 123456 | 普通用户权限 |

**UI 特性**：
- 密码可见性切换（👁️/🙈）
- 登录错误提示
- 加载状态动画

### 1.2 注册系统

**功能描述**：新用户注册账号

**学生注册**：
- 填写姓名、账号、密码、班级
- 自动分配初始宠物蛋
- 获得初始道具（苹果×3、沐浴露×2、皮球×1）

**教师注册**：
- 需要邀请码验证
- 填写任教班级
- 邀请码由管理员生成

**注册验证**：
- 账号唯一性检查
- 密码长度至少6位
- 两次密码一致性验证

## 2. 宠物系统

### 2.1 宠物类型

系统内置 **6种宠物**，每种宠物有独特的外观进化路线：

| 宠物ID | 名称 | 代表Emoji | 特点 |
|--------|------|----------|------|
| dragon | 小火龙 | 🐲 | 火焰系，攻击型 |
| cat | 星星猫 | 🐱 | 可爱系，平衡型 |
| bunny | 棉花兔 | 🐰 | 萌系，速度型 |
| bird | 彩翼鸟 | 🐦 | 飞行系，敏捷型 |
| dog | 旺财狗 | 🐶 | 忠诚系，防御型 |
| fairy | 梦精灵 | 🧚 | 魔法系，特殊型 |

### 2.2 成长阶段

宠物拥有 **7个外观阶段**，对应 21个等级：

| 等级 | 阶段名称 | 外观 | 经验范围 |
|------|---------|------|---------|
| 0 | 蛋 | 🥚 | 0 |
| 1-3 | 刚出壳 | 🐣 | 100-600 |
| 4-7 | 幼宠 | 🦎/🐱/🐰 | 1000-4200 |
| 8-11 | 成长期 | 🐲/😸/🐇 | 5600-11200 |
| 12-15 | 成熟期 | 🐉/🦁/🦊 | 13700-24000 |
| 16-18 | 进化期 | 🔥/🌟/🌸 | 28500-39000 |
| 19-20 | 传说期 | ⚡/✨/💫 | 45000+ |

### 2.3 四维状态系统

宠物拥有四个维度的状态值（0-100）：

| 状态 | 图标 | 作用 | 衰减速度 |
|------|------|------|---------|
| 生命 (health) | ❤️ | 生命值为0时宠物死亡 | 饥饿/清洁过低时下降 |
| 饱食 (hungry) | 🍗 | 影响宠物活力 | 每次 tick -2~-5 |
| 心情 (happy) | 😊 | 影响经验获取 | 每次 tick -1~-3 |
| 清洁 (clean) | 🛁 | 影响健康状态 | 每次 tick -1~-3 |

**状态影响**：
- 饱食度 < 30 或 清洁度 < 30 → 生命值下降
- 心情值高 → 经验获取加成
- 生命值 = 0 → 宠物死亡变回蛋

### 2.4 经验与升级

**经验获取方式**：

| 行为 | 经验值 | 每日上限 |
|------|--------|---------|
| 喂食 | +5 | 50/天 |
| 洗澡 | +3 | 50/天 |
| 玩耍 | +4 | 50/天 |
| 治疗 | +2 | 50/天 |

**升级公式**：
- 等级 = 根据累计经验值查表计算
- 经验表：`[0,100,300,600,1000,1500,2200,3100,4200,5600,7200,9000,11200,13700,16500,20000,24000,28500,33500,39000,45000]`

### 2.5 死亡与复活机制

**死亡条件**：
- 生命值降至 0
- 连续 7 天未喂食

**死亡后果**：
- 宠物变回蛋状态 (🥚)
- 所有状态归零
- 积分清零（惩罚机制）

**复活方式**：
- 累计喂食 3 次
- 每次喂食增加孵化进度
- 第 3 次喂食后宠物重新孵化

## 4. 任务系统

### 4.1 任务发布

**教师端功能**：

| 字段 | 说明 | 示例 |
|------|------|------|
| 任务标题 | 简短描述 | "完成数学作业" |
| 任务描述 | 详细要求 | "完成P45-P47练习题" |
| 积分奖励 | 完成后获得 | 50积分 |
| 学科分类 | 任务所属学科 | 数学/语文/英语等 |
| 截止时间 | 任务有效期 | 2026-03-14 18:00 |
| 任务图标 | 视觉标识 | 📐/📝/📚 等 |

**任务模板**：
系统预设8种任务模板，快速创建常见任务类型。

### 4.2 任务状态

学生视角的任务状态流转：

```
未领取 (pending) → 已提交 (submitted) → 已通过 (completed)
                           ↓
                     已驳回 (rejected) → 重新提交
```

| 状态 | 颜色标识 | 含义 |
|------|---------|------|
| pending | 🟡 黄色 | 未提交 |
| submitted | 🔵 蓝色 | 待审核 |
| completed | 🟢 绿色 | 已通过，积分已发放 |
| rejected | 🔴 红色 | 未通过，可重新提交 |

### 4.3 任务提交流程

1. 学生查看任务列表
2. 点击任务进入详情
3. 填写完成内容/上传证明
4. 提交等待审核
5. 教师审核通过获得积分

### 4.4 任务审核

**教师审核界面**：
- 显示学生提交内容
- 一键通过/拒绝
- 批量审核功能
- 审核历史记录

## 5. 积分系统

### 5.1 积分获取途径

| 途径 | 积分数量 | 说明 |
|------|---------|------|
| 完成任务 | 任务设定值 | 20-100积分不等 |
| 教师奖励 | 手动设定 | 课堂表现奖励 |
| 签到奖励 | 固定值 | 每日登录奖励 |

### 5.2 积分消耗途径

| 用途 | 消耗 |
|------|------|
| 购买道具 | 道具价格 |
| 离线惩罚 | 根据离线时长扣除 |

### 5.3 积分日志

记录所有积分变动：
- 时间戳
- 变动原因
- 变动数量 (+/-)
- 积分余额

### 5.4 积分与经验的关系

**重要设计**：积分与宠物经验完全解耦

- 积分仅用于购买道具
- 经验仅通过护理行为获得
- 避免"有钱就能升级"的不平衡

## 7. 离线惩罚机制

### 7.1 状态衰减 (tick)

**触发时机**：
- 页面随机间隔调用（45-90分钟）
- 学生登录时检测

**衰减规则**：
```
每次 tick：
- 饱食度：-2 ~ -5（随机）
- 心情：-1 ~ -3（随机）
- 清洁度：-1 ~ -3（随机）
- 生命值：hungry<30 或 clean<30 时 -2 ~ -4
```

### 7.2 离线惩罚

**惩罚阶梯**（基于距上次喂食的小时数）：

| 离线时长 | 积分惩罚 | 状态影响 |
|---------|---------|---------|
| < 24h | 无 | 正常衰减 |
| 24-48h | -10 | 状态补算衰减 |
| 48-72h | -30（累计） | 状态补算衰减 |
| 72-96h | -60（累计） | 状态补算衰减 |
| 96-120h | -100（累计） | 状态补算衰减 |
| 120-144h | -150（累计） | 状态补算衰减 |
| 144-168h | -200（累计） | 状态补算衰减 |
| ≥ 168h (7天) | 全部清零 | 宠物死亡 |

**死亡后果**：
- 宠物变回蛋
- 积分清零
- 状态归零
- 需要重新孵化
