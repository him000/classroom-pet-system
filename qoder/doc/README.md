```bash

# 或使用 Node.js
```

## 核心概念

### 游戏循环

```
完成任务 → 获得积分 → 购买道具 → 照顾宠物 → 宠物成长
    ↑                                              ↓
    └──────────── 持续参与学习 ←───────────────────┘
```

### 数据流向

```
用户操作 → Vue 组件 → Store 方法 → localStorage → 界面更新
```

### 三大角色

- **学生**：养宠物、做任务、看排行
- **教师**：发任务、审提交、管学生
- **管理员**：管教师、发邀请码、系统设置

## 开发注意事项

1. **纯前端架构**：所有数据存在浏览器 localStorage
2. **零构建工具**：直接编辑 HTML/JS/CSS，无需 webpack
3. **响应式设计**：适配 PC、平板、手机
4. **数据安全**：当前版本密码明文存储，生产环境需改进

## 常见问题

**Q: 数据会丢失吗？**  
A: 清除浏览器缓存会导致数据丢失，建议定期导出备份。

**Q: 支持多设备同步吗？**  
A: 当前不支持，数据仅保存在当前浏览器。

**Q: 如何添加新宠物？**  
A: 编辑 `js/data.js` 中的 `PET_TYPES` 数组。

**Q: 如何修改积分规则？**  
A: 编辑 `js/data.js` 中的 `ITEMS` 和 `GROWTH_STAGES`。
