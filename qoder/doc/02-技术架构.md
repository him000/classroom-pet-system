# 技术架构

## 架构概览

本项目采用**纯前端轻量级架构**，无需构建工具，无需后端服务器，适合快速部署和课堂演示场景。

```
┌─────────────────────────────────────────────────────────┐
│                      浏览器环境                          │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────────┐  │
│  │   Vue 3     │  │  CSS3/动画  │  │  localStorage   │  │
│  │  (CDN版)    │  │             │  │   数据持久化     │  │
│  └─────────────┘  └─────────────┘  └─────────────────┘  │
└─────────────────────────────────────────────────────────┘
```

## 技术栈详情

### 前端框架

| 技术 | 版本 | 用途 | 引入方式 |
|------|------|------|---------|
| Vue 3 | 3.x (CDN) | 前端框架、组件化、响应式数据 | CDN 全局引入 |
| CSS3 | - | 样式、布局、动画 | 本地文件 |
| JavaScript | ES6+ | 业务逻辑、API 通信 | 本地模块化 |

### 为什么选择 Vue 3 CDN 版本？

1. **零构建工具**：无需 npm、webpack、vite 等，降低使用门槛
2. **单文件运行**：一个 HTML 文件即可运行，方便课堂演示
3. **现代特性**：支持 Composition API、响应式系统、组件化
4. **学习友好**：代码直观易懂，适合教学场景

## 数据流架构

### 状态管理（Store 模式）

```
┌────────────────────────────────────────┐
│              Store (store.js)          │
│  ┌──────────────────────────────────┐  │
│  │  State (Vue.reactive)            │  │
│  │  - currentUser: 当前登录用户      │  │
│  │  - students: 学生列表             │  │
│  │  - tasks: 任务列表                │  │
│  │  - toasts: 通知消息               │  │
│  └──────────────────────────────────┘  │
│  ┌──────────────────────────────────┐  │
│  │  Actions                         │  │
│  │  - login/logout                  │  │
│  │  - addPoints/useItem             │  │
│  │  - submitTask/reviewTask         │  │
│  └──────────────────────────────────┘  │
└────────────────────────────────────────┘
           │                    │
           ▼                    ▼
    ┌─────────────┐      ┌─────────────┐
    │ localStorage │      │   Vue 组件   │
    │  持久化存储  │      │  响应式更新  │
    └─────────────┘      └─────────────┘
```

### 数据持久化策略

所有数据通过 `localStorage` 在浏览器本地持久化保存：

- **优点**：无需后端服务器，数据刷新后保留
- **限制**：
  - 清除浏览器缓存 = 数据重置为初始状态
  - 不同浏览器/设备之间数据不共享
  - 单设备容量限制（约 5-10MB）

### 组件通信方式

1. **Props Down**：父组件通过 props 向子组件传递数据
2. **Events Up**：子组件通过 `$emit` 向父组件发送事件
3. **Global Store**：共享状态通过 Store 统一管理

## 性能优化

### 已实施的优化

1. **CDN 资源**：Vue 3 使用 CDN，利用浏览器缓存
2. **CSS 变量**：统一主题管理，减少重复代码
3. **组件懒加载**：虽然未使用路由懒加载，但组件按需渲染
4. **动画优化**：使用 CSS 动画而非 JS 动画，利用 GPU 加速

### 可优化的方向

1. **图片资源**：目前使用 Emoji，如需图片建议使用 WebP 格式
2. **代码分割**：大型组件可拆分为更小模块
3. **虚拟滚动**：学生/任务列表过多时可启用

## 部署方式

### 方式一：直接打开（不推荐）

```bash
# 直接用浏览器打开
open index.html
```

⚠️ 部分浏览器对本地 `file://` 协议有限制

### 方式二：本地服务器（推荐）

```bash
# Python 3
python -m http.server 8080

# Node.js
npx serve .

# VS Code Live Server 插件
```

然后访问 `http://localhost:8080`

### 方式三：部署到静态托管

可将项目部署到任何静态托管服务：
- GitHub Pages
- Vercel
- Netlify
- 腾讯云 COS
- 阿里云 OSS
