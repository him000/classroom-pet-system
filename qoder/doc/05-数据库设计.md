# 数据库设计

## 概述

本系统使用 **localStorage** 作为数据存储方案，而非传统的关系型数据库。所有数据以 JSON 格式存储在浏览器本地。

## 存储结构

### localStorage 键值对

| 键名 | 数据类型 | 说明 |
|------|---------|------|
| `pet_system_students` | Array | 学生数据列表 |
| `pet_system_tasks` | Array | 任务数据列表 |
| `pet_system_invite_codes` | Array | 邀请码列表 |
| `pet_system_config` | Object | 系统配置（管理员账号等） |

---

## 数据模型

### 1. 学生模型 (Student)

```javascript
{
  id: Number,              // 唯一标识（时间戳生成）
  name: String,            // 真实姓名
  username: String,        // 登录账号
  password: String,        // 登录密码（明文存储，生产环境应加密）
  role: 'student',         // 角色固定为 student
  class: String,           // 班级名称
  
  // 积分系统
  points: Number,          // 当前积分
  pointsLog: Array,        // 积分变动日志
  
  // 宠物系统
  petType: String,         // 宠物类型ID (dragon/cat/bunny/bird/dog/fairy)
  petName: String,         // 宠物名字
  petExp: Number,          // 宠物经验值
  petStage: Number,        // 宠物阶段 (0-20)
  petStatus: {             // 宠物四维状态
    health: Number,        // 生命 (0-100)
    hungry: Number,        // 饱食 (0-100)
    happy: Number,         // 心情 (0-100)
    clean: Number          // 清洁 (0-100)
  },
  petDead: Boolean,        // 是否死亡
  petHatchProgress: Number,// 孵化进度 (0-3)
  
  // 背包系统
  backpack: Object,        // 道具数量 {apple: 3, soap: 2, ...}
  
  // 元数据
  joinDate: String,        // 加入日期 (YYYY-MM-DD)
  _lastFedAt: Number,      // 上次喂食时间戳
  _lastGrantReason: String,// 上次积分变动原因
  _buyDeduct: Number       // 购买扣除累计（用于弹窗提示）
}
```

#### 示例数据

```javascript
{
  id: 1,
  name: '小明',
  username: 'xiaoming',
  password: '123456',
  role: 'student',
  class: '三年一班',
  points: 320,
  pointsLog: [
    { icon: '📐', label: '完成数学作业', delta: 50, time: '2026-03-14 15:30', total: 320 }
  ],
  petType: 'dragon',
  petName: '小火龙',
  petExp: 220,
  petStage: 2,
  petStatus: { health: 75, hungry: 60, happy: 80, clean: 70 },
  petDead: false,
  petHatchProgress: 0,

  joinDate: '2026-02-01',
  _lastFedAt: 1737849600000,
  _lastGrantReason: '完成数学作业获得奖励',
  _buyDeduct: 0
}
```

### 3. 任务模型 (Task)

```javascript
{
  id: Number,           // 唯一标识（时间戳生成）
  title: String,        // 任务标题
  desc: String,         // 任务描述
  points: Number,       // 积分奖励
  icon: String,         // 任务图标Emoji
  subject: String,      // 学科分类
  deadline: String,     // 截止时间
  status: String,       // 任务状态 (active/closed)
  createdBy: Number,    // 创建者ID
  createdAt: String,    // 创建时间
  submissions: Array    // 提交记录列表
}
```

#### 提交记录结构

```javascript
{
  studentId: Number,      // 学生ID
  status: String,         // 提交状态 (pending/submitted/completed/rejected)
  content: String,        // 提交内容
  submittedAt: String,    // 提交时间
  reviewedAt: String,     // 审核时间
  resubmitted: Boolean    // 是否重新提交
}
```

## 数据安全说明

⚠️ **重要提示**：

1. **密码明文存储**：当前版本密码以明文存储在 localStorage 中
2. **数据本地存储**：所有数据仅保存在用户浏览器本地
3. **无数据同步**：不同设备/浏览器之间数据不共享
4. **清除风险**：清除浏览器缓存会导致数据丢失

**生产环境建议**：
- 使用后端服务器存储数据
- 密码使用 bcrypt 等算法加密
- 实现用户数据备份机制
- 添加数据导出/导入功能
