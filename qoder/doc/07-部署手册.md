# 部署手册

## 部署方式概览

| 方式 | 难度 | 适用场景 | 自定义域名 |
|------|------|---------|-----------|
| 本地服务器 | ⭐ 简单 | 课堂演示、本地测试 | ❌ |

| Vercel | ⭐ 简单 | 快速部署、自动更新 | ✅ |

## 方式一：本地服务器部署

### 适用场景
- 课堂演示
- 本地测试
- 无网络环境

### 部署步骤

#### 使用 Python

```bash
# 进入项目目录
cd classroom-pet-system

# Python 3
python -m http.server 8080

# 访问 http://localhost:8080
```

#### 使用 Node.js

```bash
# 安装 serve
npm install -g serve

# 启动
serve .

# 访问 http://localhost:3000
```

#### 使用 VS Code Live Server

1. 安装 Live Server 扩展
2. 右键 `index.html` → "Open with Live Server"
3. 自动打开浏览器

## 方式二：GitHub Pages 部署

### 适用场景
- 开源项目展示
- 免费静态托管
- 自动部署

### 部署步骤

#### 1. 创建 GitHub 仓库

1. 登录 [GitHub](https://github.com)
2. 点击 "New repository"
3. 填写仓库名称：`classroom-pet-system`
4. 选择 "Public" 或 "Private"
5. 点击 "Create repository"

#### 2. 推送代码

```bash
# 初始化 Git（如果尚未初始化）
git init

# 添加所有文件
git add .

# 提交
git commit -m "Initial commit"

# 关联远程仓库
git remote add origin https://github.com/你的用户名/classroom-pet-system.git

# 推送
git push -u origin main
```

#### 3. 启用 GitHub Pages

1. 进入仓库 → Settings
2. 左侧菜单选择 "Pages"
3. Source 选择 "Deploy from a branch"
4. Branch 选择 "main" / "/ (root)"
5. 点击 "Save"
6. 等待几分钟，访问显示的链接

#### 4. 自定义域名（可选）

1. 在仓库根目录创建 `CNAME` 文件
2. 写入你的域名：`pet.yourdomain.com`
3. 在域名服务商添加 CNAME 记录指向 `用户名.github.io`

## 方式三：Vercel 部署

### 适用场景
- 快速部署
- 自动持续集成
- 全球 CDN 加速

### 部署步骤

#### 1. 准备代码

确保代码已推送到 GitHub/GitLab/Bitbucket

#### 2. 导入项目

1. 登录 [Vercel](https://vercel.com)
2. 点击 "Add New Project"
3. 导入 Git 仓库
4. 选择 `classroom-pet-system` 仓库

#### 3. 配置部署

1. Framework Preset: **Other**
2. Build Command: **（留空）**
3. Output Directory: **（留空）**
4. 点击 "Deploy"

#### 4. 访问网站

部署完成后，Vercel 会提供 `.vercel.app` 域名

#### 5. 自定义域名

1. 进入项目 Dashboard
2. Settings → Domains
3. 添加你的域名
4. 按提示配置 DNS

## 方式四：腾讯云 COS 部署

### 适用场景
- 国内用户访问
- 企业级应用
- 需要备案

### 部署步骤

#### 1. 创建存储桶

1. 登录 [腾讯云控制台](https://console.cloud.tencent.com)
2. 进入对象存储 COS
3. 点击 "创建存储桶"
4. 配置：
   - 名称：`classroom-pet-system`
   - 地域：选择离你最近的
   - 访问权限：公有读私有写
5. 点击 "创建"

#### 2. 开启静态网站

1. 进入存储桶 → 基础配置
2. 找到 "静态网站" 配置
3. 开启静态网站
4. 索引文档：`index.html`
5. 错误文档：`index.html`（单页应用需要）

#### 3. 上传文件

**方式一：控制台上传**

1. 进入文件列表
2. 点击 "上传文件"
3. 选择所有项目文件
4. 等待上传完成

**方式二：COSBrowser 工具**

```bash
# 下载 COSBrowser 客户端
# 登录后拖拽上传整个文件夹
```

**方式三：CLI 命令行**

```bash
# 安装 coscmd
pip install coscmd

# 配置
coscmd config -a <SecretId> -s <SecretKey> -b classroom-pet-system -r ap-guangzhou

# 上传
coscmd upload -r ./ classroom-pet-system/
```

#### 4. 访问网站

使用提供的默认域名访问，或绑定自定义域名

#### 5. 绑定自定义域名

1. 进入存储桶 → 域名管理
2. 点击 "自定义源站域名"
3. 输入你的域名
4. 在域名服务商添加 CNAME 记录
5. 等待 DNS 生效

## 方式五：阿里云 OSS 部署

### 适用场景
- 国内用户访问
- 阿里云生态
- 需要备案

### 部署步骤

#### 1. 创建 Bucket

1. 登录 [阿里云控制台](https://oss.console.aliyun.com)
2. 点击 "创建 Bucket"
3. 配置：
   - 名称：`classroom-pet-system`
   - 地域：选择最近的
   - 存储类型：标准存储
   - 读写权限：公共读
4. 点击 "确定"

#### 2. 配置静态网站托管

1. 进入 Bucket → 基础设置
2. 找到 "静态页面"
3. 默认首页：`index.html`
4. 默认 404 页：`index.html`

#### 3. 上传文件

**方式一：控制台上传**

1. 进入文件管理
2. 点击 "上传文件"
3. 选择所有文件

**方式二：ossutil 工具**

```bash
# 下载 ossutil
# 配置
ossutil config

# 上传
ossutil cp -r ./ oss://classroom-pet-system/
```

#### 4. 访问网站

使用 Bucket 域名访问，或绑定自定义域名

## 部署后配置

### 修改默认账号密码

**重要**：部署后应立即修改默认账号

1. 使用管理员账号登录
2. 进入系统设置
3. 修改管理员账号和密码

### 配置邀请码

1. 管理员登录
2. 进入邀请码管理
3. 生成新的教师邀请码
4. 删除默认邀请码

### 数据备份策略

由于使用 localStorage 存储数据，建议：

1. **定期提醒用户导出数据**
2. **实现数据导出功能**（可添加按钮导出 JSON）
3. **考虑后端版本**（如需数据持久化）

## 故障排查

### 页面空白

**原因**：资源路径错误

**解决**：
- 检查 `index.html` 中的路径是否为相对路径
- 确保所有资源文件都已上传

### 样式丢失

**原因**：CSS 文件未加载

**解决**：
- 检查 `styles/` 目录是否正确上传
- 检查浏览器控制台是否有 404 错误

### 功能异常

**原因**：浏览器兼容性问题

**解决**：
- 检查浏览器版本是否符合要求
- 查看控制台错误信息

### 数据丢失

**原因**：localStorage 被清除

**解决**：
- 提醒用户定期导出数据
- 考虑添加数据导出/导入功能

## 安全建议

### 生产环境注意事项

1. **修改默认密码**
   - 管理员：admin / admin888
   - 教师：teacher / teacher123
   - 学生：xiaoming / 123456

2. **启用 HTTPS**
   - 所有部署方式都应使用 HTTPS
   - 防止数据被窃取

3. **定期备份**
   - 导出 localStorage 数据
   - 保存到安全位置

4. **访问控制**
   - 如使用云存储，设置合适的访问权限
   - 避免敏感文件被公开访问

## 性能优化

### 部署前优化

1. **压缩资源**
   ```bash
   # 压缩 CSS
   npx clean-css-cli styles/main.css -o styles/main.min.css
   
   # 压缩 JS（可选，因为代码量不大）
   ```

2. **启用 Gzip**
   - 大部分托管服务默认启用
   - 可减少 60-80% 传输大小

3. **启用 CDN**
   - 使用托管服务提供的 CDN
   - 加速全球访问

### 监控

1. **访问统计**
   - 使用百度统计 / Google Analytics
   - 了解用户访问情况

2. **错误监控**
   - 使用 Sentry 等错误追踪服务
   - 及时发现线上问题
